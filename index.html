Mô hình Von Neumann

Nền tảng của hầu hết máy tính hiện nay được John von Neumann đề xuất năm 1945.

  • CPU: Bộ xử lý trung tâm gồm ALU và Control Unit.
  • Memory: Bộ nhớ duy nhất chứa cả lệnh và dữ liệu.
  • Bus: Hệ thống kết nối các thành phần.

Chu kỳ thực thi Lệnh

1. Fetch (Lấy lệnh)

CPU lấy mã lệnh từ bộ nhớ chính dựa trên địa chỉ trong Program Counter (PC).

2. Decode (Giải mã)

Control Unit phân tích mã máy để biết CPU cần làm gì (Cộng, Nhân, Nhảy...).

3. Execute (Thực thi)

ALU thực hiện tính toán vật lý hoặc điều khiển luồng dữ liệu.

4. Store/Write-back

Kết quả được ghi lại vào Registers hoặc Memory để sử dụng tiếp.

Kiến trúc Tập lệnh (ISA)

RISC

Reduced Instruction Set Computer

  • Lệnh ngắn, chiều dài cố định.
  • Thực hiện 1 lệnh trong 1 chu kỳ.
  • Ít lệnh nhưng chạy rất nhanh.
  • Ví dụ: ARM (Apple M1/M2, Smartphone).

CISC

Complex Instruction Set Computer

  • Lệnh phức tạp, chiều dài thay đổi.
  • Một lệnh có thể làm nhiều việc.
  • Tiết kiệm bộ nhớ cho code.
  • Ví dụ: Intel x86 (PC, Laptop).
